# ShopEnhancement (STS2 Mod)

本项目为《Slay the Spire 2》模组，旨在增强商店与奖励系统的策略深度，提供更多选择与趣味性，同时保持游戏平衡。
Base By https://github.com/Alchyr/ModTemplate-StS2

前置依赖: [BaseLib-StS2](https://github.com/Alchyr/BaseLib-StS2)

## 更新日志

### v0.4 (最新)

  - 从自己的长期牌堆（Deck）中选择卡牌传给随机队友。
  - 支持“传牌数量范围、基础费用、每张递增费用”配置。
- **平衡参数优化（默认值）**：
  - `EnchantStartShopVisit`: 4
  - `EnchantReplaceChance`: 0.30
  - `EnchantCost`: 105
  - `EnchantAmountRange`: 1~2
  - `EnchantCardCountRange`: 1~2
  - `GiftServiceCardCountRange`: 1~1
  - `GiftServiceBaseCost`: 85
  - `GiftServiceStepCost`: 55

## 功能特性 (默认配置)

本模组包含以下核心功能调整：

### 1. 商店卡牌移除优化
- **更灵活的费用**：初始移除费用调整为 **50 金币**（原版 75），后续每次增加 25 金币。鼓励玩家早期精简卡组。
- **多次移除**：每进入一次商店，最多可移除 **3 张** 卡牌（需支付递增费用）。

### 2. 商店刷新机制
- **刷新商品**：花费 **40 金币** 刷新商店内的所有卡牌、遗物和药水。
- **次数限制**：每家商店限刷新 **3 次**，防止过度滥用。

### 3. 经济补偿机制
- **未购物奖励**：如果离开商店时未购买任何物品，将获得 **15 金币** 的路费补偿。
- **跳过卡牌奖励**：战斗胜利后，若选择跳过卡牌奖励，将获得 **15 金币**。

### 4. 跨职业卡牌
- **更多样化的构筑**：商店中的卡牌有 **20%** 的概率被替换为其他职业的卡牌，为构筑带来意外惊喜与全新思路。

### 5. 全内容解锁
- **一键解锁**：进入主菜单时自动解锁所有卡牌、遗物、药水以及**全部角色**（通过揭示所有纪元实现）。无需繁琐的刷图过程，直接体验游戏全部内容。

### 6. 出售模式 (Sell Mode)
- **回收资源**：在商店界面点击“开启出售模式”按钮后，**右键点击** 遗物或药水即可出售。
- **定价机制**：
    - **遗物**：回收价格为原价的 **35%**。特殊遗物（Boss/初始/事件）有固定的基础回收价。
        - *策划备注：Boss 遗物现在非常值钱，但事件遗物（如通过负面事件获得的）回收价较低。*
    - **药水**：回收价格为原价的 **25%**。
        - *策划备注：药水作为消耗品，不应成为主要经济来源。*
    - 设有最低回收价保障（遗物 30 金币，药水 15 金币）。
### 7. 送礼模式 (Gift Mode)
- **分享财富**：在多人联机游戏中，你可以花费金币购买卡牌并直接赠送给队友。
- **简单操作**：
    - 点击商店界面的 "Gift Mode" 按钮开启。
    - 点击目标按钮切换想要赠送的队友。
    - 像往常一样购买卡牌，它会直接飞入队友的牌组中！
- **视觉反馈**：接收者会看到提示，并看到卡牌飞入牌组的动画。

### 8. 商店服务槽位（移除 / 附魔 / 随机队友传牌）
- **服务切换规则**：
    - 从第 **4** 次进店起，服务位有 **30%** 概率切换为特殊服务。
    - 特殊服务会在“附魔服务”与“随机队友传牌服务”之间择一，并在当前商店内固定。
- **附魔服务默认值**：
    - 附魔基础费用 **105**，强度范围 **1~2**，可附魔数量范围 **1~2**。
- **随机队友传牌默认值**：
    - 传牌数量范围 **1~1**（默认每次传 1 张长期牌堆卡牌）。
    - 费用为基础 **85**，每增加 1 张额外加价 **55**。

## 构建与安装

在项目根目录执行：

```powershell
dotnet publish -c Release
```

构建成功后，Mod 文件会输出到 `public/ShopEnhancement`：
- `ShopEnhancement.dll`
- `ShopEnhancement.pck` (如果有资源)
- `mod_manifest.json`

可以用 `-p:ModOutputDir=<path>` 覆盖本地输出目录。
如需导出 `ShopEnhancement.pck`，还需要传入 `-p:GodotPath=<Godot 控制台程序路径>`。
## 直接安装

见右侧 releases 。

将编译后的 `ShopEnhancement.pck & dll` 放入游戏 `mods` 目录。

**注意：** 你还需要安装前置依赖 [BaseLib-StS2](https://github.com/Alchyr/BaseLib-StS2)。
请从其 Release 页面下载 `BaseLib.dll` 和 `BaseLib.pck` 并一同放入 `mods` 目录。

## 目录结构

- `ShopEnhancement/`：模组主要逻辑与补丁代码
- `ShopEnhancementConfig.cs`：核心数值配置文件
- `mod_manifest.json`：模组元数据

## 备注

- 模组通过 Harmony 注入逻辑，兼容性取决于游戏版本。
- 若导出失败或游戏提示未加载，先检查 `public/ShopEnhancement` 是否包含 `ShopEnhancement.dll`、`ShopEnhancement.pck` 与 `mod_manifest.json`，再将该目录复制到游戏 `mods` 目录进行本地测试。
